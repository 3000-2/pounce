# Pounce

마우스 없이 키보드로 macOS 화면의 아무 곳이나 클릭하고 스크롤하는 도구. 단축키를 누르면 클릭 가능한 요소 위에 힌트 라벨이 뜨고, 라벨을 입력하면 그 요소가 클릭된다.

## 사용법

### 힌트 클릭 — `⌥⇧F`

1. 단축키를 누르면 앞 앱의 클릭 가능한 요소에 파란 힌트 칩이 표시된다
2. 칩의 글자를 입력하면 해당 요소를 클릭한다 — 후보가 여러 개 남아 있는 동안엔 각 요소에 윤곽선이 표시된다
3. `Esc` 취소 · `Delete` 한 글자 지우기

힌트가 떠 있는 동안에도 `⌘C` 같은 단축키는 평소처럼 동작하고, 마우스를 클릭하거나 앱을 전환하면 힌트는 자동으로 사라진다.

### 스크롤 — `⌘⇧S`

가장 큰 스크롤 영역에 파란 테두리가 표시되고, 키보드로 스크롤한다.

| 키 | 동작 |
| --- | --- |
| `j` / `k` | 아래 / 위 |
| `h` / `l` | 왼쪽 / 오른쪽 |
| `d` / `u` | 반 페이지 아래 / 위 |
| `g` / `G` | 맨 위 / 맨 아래 |
| `Tab` | 다음 스크롤 영역으로 전환 |
| `Esc` | 종료 |

### 메뉴바

- **설정…**: 힌트/스크롤 단축키를 원하는 키 조합으로 변경 (필드 클릭 후 새 조합 입력, 재시작 후에도 유지)
- **로그인 시 열기**

## 설치

```sh
brew install --cask --no-quarantine 3000-2/tap/pounce
```

`--no-quarantine`은 아직 공증되지 않은 앱의 Gatekeeper 차단을 피하기 위한 플래그다. 빼고 설치했다면 첫 실행 시 시스템 설정 → 개인정보 보호 및 보안에서 "그래도 열기"를 한 번 눌러야 한다.

소스 빌드: `git clone https://github.com/3000-2/pounce.git && cd pounce && make run`

첫 실행 시 **시스템 설정 → 개인정보 보호 및 보안 → 손쉬운 사용**에서 Pounce를 켜고 앱을 다시 실행한다.

지원: macOS 14+. 네이티브 앱, Electron 앱(Slack, VS Code 등), Chromium 브라우저의 웹 콘텐츠까지 동작한다.

## 문제 해결

- **힌트가 안 뜸**: 손쉬운 사용 권한 확인. 켜져 있는데도 안 되면 `tccutil reset Accessibility com.poc.pounce` 후 다시 부여.
- **Chrome에서 첫 트리거가 빈손**: 페이지 접근성 트리가 만들어지는 데 1~2초 걸릴 수 있다. 한 번 더 트리거하면 된다.

## 개발

`make test`로 유닛 테스트, `make bundle`로 앱 번들 생성. 에이전트/기여 규칙은 `AGENTS.md`.
